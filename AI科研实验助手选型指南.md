# AI 科研实验助手 · 端到端操作白皮书

> **适用场景**：做基础实验（生化/材料/动物实验等）时需要查询实验参数和步骤，担心通用 AI 编造信息，希望每个参数都有文献出处，并能根据自己实验内容调整前人方案。
>
> **核心诊断**：你的问题不是"AI 不够好"，而是**输入层（论文检索）和输出层（结构化加工）断裂了**。通用 AI 只有输出层、没有实时检索能力——这就是幻觉的根源。这份白皮书给你一套两层缝合方案。

---

## 一、问题诊断：两层的断裂

### 1.1 四个表层痛点

| # | 痛点 | 本质 |
|---|------|------|
| ① | **领域陌生期**：不了解参数范围，无从下手 | 信息不对称 |
| ② | **AI 幻觉焦虑**：担心 AI 编造参数 | LLM 概率生成，不校验事实 |
| ③ | **可追溯性缺失**：参数没有文献出处 | 无法写入论文/实验记录 |
| ④ | **参数个性化调整**：需要按自己实验适配前人方案 | 需要"参数映射"而非"参数复制" |

### 1.2 一个深层的根因

这些问题指向同一个根因——**科研 AI 需要两层协作，但大多数人只配了一层**：

```
┌──────────────────────────────────────────────────┐
│                端到端科研 AI 架构                  │
├──────────────────────────────────────────────────┤
│                                                   │
│  🔵 输入层 · 文献检索 ← 大多数人缺这一层            │
│  职责：实时搜索真实论文库，把原始文献"拉进上下文"    │
│  工具：Consensus / Methods Muse / PubCompare       │
│  ↓ 产出：PDF全文 + 结构化参数 + DOI                │
│                                                   │
│  🟢 输出层 · 结构化加工 ← 大多数人用通用AI在这一层   │
│  职责：精读文献、提取参数、标注出处、设计方案         │
│  工具：Claude Project（配 Instructions）            │
│  ↓ 产出：参数表(标证据级) + 实验方案 + 文献卡片      │
│                                                   │
│  🔴 质检层 · 防幻觉兜底 ← 两层都跑完再过            │
│  职责：交叉验证、溯源核对                            │
│                                                   │
└──────────────────────────────────────────────────┘

断裂的后果：
  ❌ 只有输出层 → AI 靠"回忆"回答 → 编造参数、假DOI、无法溯源
  ✅ 两层完整   → 先搜真实论文 → 再从论文里提取参数 → 每条有出处
```

### 1.3 一个问题试出你的架构是否完整

问任何一个 AI 工具这个问题，看它能不能做到**参数 + 文献出处 + DOI**三要素齐全：

> "微流控制备磁性水凝胶微球，油相:水相流速比的建议值是多少？给出文献出处。"

| 只有输出层（通用 AI） | 两层完整（缝合方案） |
|------|------|
| "建议流速比 1:3 到 1:10..."（无出处） | "1:4，来自 Chen et al. 2023, Lab Chip, DOI: 10.1039/..." |
| 参数可能是"合理想象" | 参数来自真实论文 Methods 段落 |

## 三、输入层深入：外部工具链

### 3.1 为什么 Claude Project 不能替代这一层

Claude Project 的 Instructions 写得再好（即使是第4章拆解的那份顶尖配置），也无法绕过这个硬约束：

> **Claude Project 没有实时论文检索引擎。** 它能精读你上传的 PDF，但不能自己去 PubMed/Consensus 搜论文。

这意味着：Project 内的 Claude 只能在**你喂给它的文献范围内**给出 📚 级答案。你没传 PDF 的方向，它只能凭训练数据回答 → 最多标 🧩 或 ⚗️ → 这就是他最初焦虑的根源。

### 3.2 三件套能力定位

| 工具 | 数据规模 | 核心能力 | 在架构里的角色 | 价格 |
|------|:--:|------|------|------|
| **[Consensus](https://consensus.app)** | 2.2亿篇论文 | 跨学科全景搜索 + 参数溯源 | **输入层主力**：课题维度的文献全覆盖 | 免费层 |
| **[Methods Muse](https://methods.springernature.com/)** | 10万+ 同行评审协议 | 单学科协议生成 + 故障排查 | **输入层补充**：Methods 段落的精细化检索 | 免费 beta |
| **[PubCompare.ai](https://www.pubcompare.ai)** | 4000万 论文+专利 | 协议参数横向对比 | **输入层收尾**：多篇论文参数抽取为共识表 | 商业 |

### 3.3 按课题类型选输入起点

| 课题类型 | 举例 | 先从哪开始 | 再用什么 |
|----------|------|:--:|------|
| **单学科单步实验** | Western blot、PCR、细胞培养 | Methods Muse | Consensus（交叉验证） |
| **多学科交叉课题** | 微流控+材料+动物实验 | **Consensus**（全景扫描） | Methods Muse（拆子问题查） |
| **多方案参数对比** | 选栓塞材料、选动物模型 | Consensus → PubCompare | — |

> ⚠️ **核心原则**：跨学科课题不要从 Methods Muse 开始——它的协议库偏生物/化学，材料/工程领域覆盖不如 Consensus。先让 Consensus 扫出 5 篇综述拿到全景，再按子问题用 Methods Muse 深挖。

## 四、输出层深入：Claude Project 配置

> 以下内容基于一份实际生产中的 Claude Project Instructions（340 行完整配置），经本人授权分享分析。这份配置的完整文件可供参考，核心要点如下。

### 4.1 这份配置好在哪——五个设计亮点

| # | 模块 | 做了什么 | 为什么重要 |
|---|------|---------|------------|
| 1 | **证据分级标注** | 📚文献实证 / 🧩类比推断 / ⚗️待优化起点 / ⚠️需查证——四级强制标注 | **这是防幻觉的规则层核心**。没有这个机制，Claude 会把推断伪装成事实。有它之后，每次给参数都必须自报"可信度级别" |
| 2 | **双模式分流** | 有 PDF 附件 → 自动进入文献精读模式；无 PDF → 自动进入课题顾问模式 | 避免拿通用知识冒充文献内容。上传 PDF 前后的回答质量天差地别 |
| 3 | **精读 SOP（B1→B2→B3）** | B1 段落精读（中英对照+科学解读+课题转化）→ B2 全文总结（含审稿式点评）→ B3 知识库卡片（Artifact） | 把一篇论文从"读过"变成"可检索的结构化知识块"。读完 5 篇后，5 张卡片组成一个微型知识库 |
| 4 | **实验设计模板（C1）** | 假设→分组→参数表(每行强标证据级)→统计设计→表征手段→失败规避→出处汇总 | 每次问实验参数，输出天然带证据级别。不给裸参数 |
| 5 | **课题基线预载** | 把 Project A/B 的目标参数写在 Instructions 里，每次回答自动关联 | 避免 AI 给出与课题目标矛盾的参数建议 |

### 4.2 证据分级系统——防幻觉的规则核心

这是这份配置里最有价值的机制。一个参数被给出时，必须标注它处于哪个级别：

| 标注 | 含义 | 例子 |
|:--:|------|------|
| 📚 文献实证 | 直接来自已发表文献，附作者/期刊/年份 | "流速比 1:4，Chen et al. 2023, Lab Chip" |
| 🧩 类比推断 | 从相似体系推理得出，需实验验证 | "参考 PLGA 微球的制备参数外推至 GelMA 体系" |
| ⚗️ 待优化起点 | 建议的预实验起始值，需梯度优化 | "建议从 5 wt% 起始，按 2.5 wt% 梯度递增" |
| ⚠️ 需查证 | 不确定/原文未提供/记忆可能有误 | "此数值我不确定，建议在 PubMed 用关键词 XX 复核" |

> 🚨 **关键规则**：任何具体浓度/温度/流速/粒径，**只有带 📚 出处时才可作为事实陈述**；否则一律标 ⚗️ 或 🧩，并声明"需预实验优化，不得作为定论"。

### 4.3 这份配置的唯一缺口

在 C 模式（课题顾问模式）下，当 Project 内没有相关文献 PDF 时：

```
用户问："微流控流速比建议值？文献出处？"

Claude 能做：
  ✅ 根据训练数据给一个范围 → 标 ⚗️
  ✅ 诚实标注"我不确定" → 标 ⚠️
  ❌ 去 PubMed 实时搜最新论文
  ❌ 给参数标 📚 + 具体 DOI

→ 参数表里大部分行只能填 ⚗️——规则兜底了，但没解决问题
```

**补丁方案**：在 Instructions 的 C 模式入口处加一段**检索前置提示**：

```markdown
## 🆕 检索前置判断（C 模式激活时自动执行）

当用户提出实验问题且 Project 内无该方向文献 PDF 时，先输出：

> ⚠️ 当前 Project 内无该方向文献，建议先检索再问我：
> 1. 打开 Consensus (consensus.app) → 搜索 `[自动生成关键词]`
> 2. 筛选被引最多的 5 篇综述 + 3 篇原创 → 下载 PDF
> 3. 上传到本 Project → 我执行 B1/B2/B3 精读
> 4. 再输 `实验` 触发 C1 → 每个参数都能标 📚
>
> 要我先生成英文检索关键词吗？

如果用户坚持直接给建议，则正常执行 C 模式，
但所有参数强制标 ⚗️ 或 🧩，不得标 📚。
```

**这段补丁的作用**：把"你需要先搜文献"从一个外部建议变成 Project 内部的工作流步骤——Claude 会在合适时机自动提醒，用户不需要自己记住"先去 Consensus 搜"。

## 五、缝合工作流：端到端的五个阶段

### 5.1 流程图

```
阶段1 · 全景扫描          阶段2 · 协议深挖
┌──────────────┐       ┌──────────────┐
│  Consensus   │       │ Methods Muse │
│  搜课题全貌   │       │  拆子问题查   │
│  拿5篇综述   │       │  拿标准协议   │
└──────┬───────┘       └──────┬───────┘
       │                      │
       └────────┬─────────────┘
                ▼  PDF + 协议 + 参数范围
       ┌──────────────────────────────┐
       │      阶段3 · 文献精读         │
       │  上传 PDF 到 Claude Project  │
       │  B1段落精读 → B2全文总结      │
       │  → B3知识库卡片               │
       └──────────────┬───────────────┘
                      ▼  每篇论文变成结构化卡片
       ┌──────────────────────────────┐
       │      阶段4 · 实验设计         │
       │  触发 C1 课题顾问模式         │
       │  参数表现在能填 📚 + DOI      │
       └──────────────┬───────────────┘
                      ▼
       ┌──────────────────────────────┐
       │      阶段5 · 质检收尾         │
       │  追问引用 + 交叉验证 + 溯源核对│
       └──────────────────────────────┘
```

### 5.2 逐步详解（每一步标注原因）

| 阶段 | 操作 | 工具 | **为什么这一步必须做（不可跳过）** |
|:--:|------|:--:|------|
| **1** | 用英文关键词搜课题全貌，按 Citations 排序，找到被引最多的 5 篇综述 + 3 篇原创论文 | Consensus | Claude Project 没有论文检索能力。**不先做这一步，后面 C1 参数表"出处"列全部只能填 ⚗️ 或 ⚠️**。Consensus 搜 2.2 亿篇论文，决定了后面所有参数能否标 📚。选 5+3 是因为：<5 篇跨学科覆盖不全，>10 篇 Project 上下文可能溢出 |
| **2** | 逐个子问题查协议（微流控参数/材料表征/动物模型/自黏附测试）| Methods Muse | Consensus 偏宏观搜索，对 Methods 段落的具体操作步骤覆盖不够细。**Methods Muse 专门检索协议和方法段落**，补齐具体参数的精度 |
| **3** | 可选的参数横向对比 | PubCompare | 如果多篇论文对同一参数值不一致（如流速比有 1:3 和 1:6 两种），**PubCompare 自动生成共识表 + 标记异常值**。手动 Excel 对比慢 5 倍且易漏 |
| **4** | 把 PDF 上传到 Claude Project | Claude Project | **这是缝合的关键动作**。PDF 进 Project 后，Claude 能从原文提取参数，C1 才能给出 📚 级证据。不上传 PDF 的话，Project 配置再好也只是在"想象" |
| **5** | 每篇执行 B1→B2→B3 精读 | Claude Project（工作流B）| B1 精读把 Methods 段落的具体数值（流速/浓度/温度/时间）精确提取出来。B3 卡片把每篇论文变成可检索的结构化知识块——读 5 篇得 5 张卡片，组成一个微型课题知识库 |
| **6** | 触发 C1 实验设计咨询 | Claude Project（工作流C）| 此时 Project 已有 5-8 篇论文全文+卡片，**C1 参数表能从 PDF 原文直接引用数值并标注段落**。之前只能填 ⚗️/🧩 的参数现在能填 📚 了。证据级别从"建议值"变成"文献值" |
| **7** | 三步防幻觉质检 | 人工 | ① 追问引用：每个 📚 参数的 DOI 能被打开 ② 交叉验证：关键参数在 Consensus 和 Methods Muse 结果中一致 ③ 溯源核对：关键数值回到论文 Methods 段落确认。**这是最后一道防线，即使前六步都跑对，这里也不能省** |

## 六、实战走通：微流控制备磁性凝胶微球治疗动脉瘤

> 以下把五阶段缝合工作流在一个真实课题上完整走一遍。每步标注了输入、操作、产出。

### 课题信息

- **Project A**：微流控制备两种互补反应性磁性水凝胶微球 → 混合后自黏附 → 封堵动脉瘤
- **Project B**：微流控制备磁性 Janus 自黏附水凝胶纤维用于血管封堵
- **课题基线**：微球直径 50-150 μm / 磁性填料 10-40% / 外加磁场 50-300 mT

### 阶段1 · 文献全景扫描

```
工具：Consensus（免费层）
输入：microfluidic synthesis magnetic hydrogel microsphere aneurysm embolization
操作：按 Citations 排序 → 取前5篇综述 + 前3篇高引原创论文

产出：
  综述 #1: "Embolic agents for endovascular treatment of cerebral aneurysms"
          (2023, 180+ citations) — 材料对比表
  综述 #2: "Microfluidic synthesis of functional hydrogel microparticles"
          (2024, 90+ citations) — 流速比/浓度参数范围
  综述 #3: "Magnetic nanoparticles in biomedical applications: A review"
          (2022, 450+ citations) — Fe₃O₄/PLGA/GelMA 对比
  综述 #4: "Animal models of intracranial aneurysms"
          (2021, 120+ citations) — 兔/犬模型造模方案对比
  综述 #5: "Self-healing and self-adhesive hydrogels"
          (2023, 80+ citations) — 黏附化学机制对比表
  论文 A-D: (4篇原创，含具体参数值)

✅ 检查点：5篇综述 + 4篇论文的 PDF 都在本地了
```

### 阶段2 · 协议深挖

```
工具：Methods Muse
操作：4 次独立查询，每次针对一个子问题

子问题 A — 微流控：
  输入：microfluidic flow-focusing droplet generation flow rate ratio
        magnetic nanoparticle-loaded hydrogel
  产出：油相:水相 1:2~1:10；芯片材质 PDMS/玻璃；CV<5%

子问题 B — 材料表征：
  输入：standard characterization magnetic hydrogel SEM TEM VSM rheology
  产出：SEM(形貌)、TEM(粒径)、VSM(磁响应曲线)、流变仪(G')、溶胀率、FTIR、TGA

子问题 C — 动物模型：
  输入：rabbit carotid sidewall aneurysm model embolization protocol
  产出：兔颈总动脉侧壁瘤（标准方案）+ 术后 4/8/12w DSA 随访

子问题 D — 黏附测试：
  输入：self-adhesive hydrogel lap shear test two-component
  产出：搭接剪切测试（ASTM F2255）+ 30s-5min 黏附力-时间曲线

✅ 检查点：4 个子问题都有协议 + 参数范围
```

### 阶段3 · 文献精读

```
工具：Claude Project（工作流 B1→B2→B3）
操作：逐篇上传 PDF → 输入"精读"

对综述 #1 执行 B2 全文总结：
  → 提取 Table 2 材料对比（海藻酸钙/PLGA/GelMA/壳聚糖/PEGDA/PVA）
  → 四维评级：⭐⭐⭐⭐ | 高 | 强 | 🟢核心参考

对论文 A 执行 B1 段落精读 + B3 卡片：
  → Methods 段落提取：流速比 1:4, Fe₃O₄ 5 wt%, 交联时间 30 min
  → 卡片存为 Artifact，可检索

……对 9 篇论文逐一处理

✅ 检查点：9 张文献卡片在 Project 内可检索
```

### 阶段4 · 实验设计

```
工具：Claude Project（工作流 C1）
输入（在 Project 内，所有 PDF 和卡片已就位）：
```

## 八、常见问题

### Q1：这个方案需要付费吗？
日常不需要。Consensus 免费层每天可搜几十次，Methods Muse 免费 beta，Claude Project 需要 Claude 订阅但你不是已经有了。PubCompare 按需。

### Q2：如果课题比较小（比如就一个 WB 条件优化），也需要跑完五个阶段吗？
不需要。单步实验缩短版：Methods Muse 直接查协议（阶段2）→ 追问出处（阶段5）。两个步骤就够。

### Q3：中文提问可以吗？
Consensus 和 Methods Muse 的英文搜索精度远高于中文，**强烈建议用英文关键词**。不会翻的话，让 Claude Project 里的 Claude 帮你把中文问题翻译成英文检索词。

### Q4：如果找不到完全匹配的论文怎么办？
正常的。① 放宽搜索词，用更通用的术语 ② 找最接近的论文，Methods 段落往往有相似体系可参考 ③ 把"完全匹配"的期望调整为"最接近参考 + 自己微调"。
