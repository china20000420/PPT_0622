## 1. 核心思路

论文将文档拼接重新表述为**装箱问题 (Bin Packing Problem, BPP)**。

将文档切分为长度不超过 L 的 chunk。目标是：用最少的 bin（训练序列）装下所有 chunk，每个 bin 容量为 L。

```
min  M
s.t. Σ_{c∈s_j} l(c) ≤ L,  ∀j
     s_1 ∪ ... ∪ s_M = C,  s_i ∩ s_j = ∅
```

BPP 是 Karp 1972 年证明的 NP-Hard 问题。论文采用 **Best-Fit-Decreasing (BFD)** 启发式算法近似求解。

---

## 2. 算法流程

### Phase 1 — Segmentation（文档预切分）

| 条件 | 操作 |
|------|------|
| l(d) ≤ L | 文档完整保留为 1 个 chunk，不做切分 |
| l(d) > L | 切分为 ⌈l/L⌉ 个 chunk：[L, L, ..., r]，r = l mod L |

仅约 5% 的文档（真正长度 > L 的）被切分——这是"必要截断"，受限于模型上下文长度。其余 95%+ 完整保留。

### Phase 2 — BFD Packing（装箱）

**步骤 (a)：降序排列**

所有 chunk 按长度从大到小排序。因为 l(c_i) ∈ [1, L] 为有限整数，使用**计数排序 (Counting Sort)**，复杂度 O(N)，而非比较排序的 O(N log N)。

**步骤 (b)：BFD 贪心放置**

依次处理降序排列中的每个 chunk c：
- 在所有已有 bin 中，找剩余容量 ≥ l(c) 且**剩余容量最小**的 bin（Best-Fit 准则）
- 若找到 → 放入该 bin
- 若所有 bin 的剩余容量都 < l(c) → 创建新 bin

Best-Fit vs First-Fit：Best-Fit 选"最满但还能容纳的 bin"，保留大容量 bin 给后续大 chunk，减少新 bin 创建。

**步骤 (c)：线段树加速搜索**（核心性能优化）

BFD 的关键性质：**不区分相同剩余容量的 bin**。两个都剩 5 tokens 的 bin，对 BFD 完全等价——只需知道"有没有容量为 5 的 bin"，不需要区分具体哪个。

因此只需为 [1, L] 这 L 种容量值维护数据结构，而非为 N 个 bin。

- **线段树结构**：L 个叶节点对应容量 1..L。叶节点值 = 该容量值（若存在），否则 = -∞。内部节点 = max(左子, 右子)
- **查询**：从根出发，若左子最大值 ≥ l(c) → 进左，否则 → 进右。到达叶节点时，索引 = Best-Fit 容量。O(log L)
- **更新**：放入 chunk 后，旧容量叶设为 -∞，新容量叶设为新值。更新从叶到根路径。O(log L)

### 复杂度总结

```
排序：O(N)           （计数排序，利用 l(c)∈[1,L] 的有界性）
装箱：O(N log L)     （N 次查询 × O(log L) + N 次更新 × O(log L)）
总计：O(N log L)     （L=8192 时 log₂L=13，近乎线性）
```

## 3. 装箱示例

L=8，10 个 chunk：{6, 4, 4, 3, 2, 2, 1, 7, 5, 8}，Σ=42 tokens。

```
降序: [8, 7, 6, 5, 4, 4, 3, 2, 2, 1]
装箱过程:
  8→B1[0]      7→B2[1]      6→B3[2]      5→B4[3]
  4→B5[4]      4→B5[0]  ← 完美放入
  3→B4[0]  ← Best-Fit: 剩余恰好 3
  2→B3[0]      2→B6[6]      1→B2[0]
M=6 bins, 利用率 87.5%
理论最优: OPT = ⌈42/8⌉ = 5
BFD 渐近保证: M ≤ (11/9)·OPT + 4 (Johnson et al., 1974)
```

## 6. 方法定位

**直接回应随机拼接"缺陷一（截断）"**：不改文档顺序、不改序列长度，仅修改装入方式。

**四种方法中投入产出比最优**：填充 < 0.003%（零代价），22 任务全正向无退化（收益最全面）。

**已集成 HuggingFace TRL**：strategy="ffd"，替换数据准备的 packing 函数即可。

**局限**：不区分语义——无关文档可同箱。与 ICLM 完全正交，可组合使用（先装箱后排序）。
