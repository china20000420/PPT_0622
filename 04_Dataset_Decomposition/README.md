## 1. 核心思路

前面三种方法都在回答"如何把变长文档拼成固定长度 L 的序列"。DD 问了一个更根本的问题：**L 为什么必须固定？**

DD 不做拼接，而是把每篇文档按 token 长度的二进制展开**主动分解**为多个 2 的幂次长度的连续片段，各片段按长度分别入桶。训练时从不同桶采样，序列长度可变、batch size 相应调整，总 token 数恒定。配合课程策略，以短序列低成本训练打底，长序列精训拔高。

## 2. 算法流程

### 2.1 二进制分解（Binary Decomposition）

对每篇文档 d，取其 token 长度 l，做二进制展开：

```
l = Σ_{k=1}^{K} 2^{i_k},    i_1 > i_2 > ... > i_K ≥ 0
```

将文档切分为 K 段连续序列，长度分别为 2^{i₁}, 2^{i₂}, ..., 2^{i_K}。

长度为 2^i 的片段放入桶 D_i。论文桶范围：i ∈ {8, ..., 13} = {256, 512, 1024, 2048, 4096, 8192}。

**示例**：
- l = 2200 = 2048 + 128 + 16 + 8 → D₁₁, D₇, D₄, D₃
- l = 5600 = 4096 + 1024 + 256 + 128 + 64 + 32 → D₁₂, D₁₀, D₈, D₇, D₆, D₅

**为什么是 2 的幂次？** 四个理由：
1. GPU 矩阵乘法对齐（cuBLAS tile = 128/256）
2. 二进制展开分段数最少（贪心最优）
3. FlashAttention 块优化兼容
4. 桶数可控（仅 6 个），每桶序列充足，采样稳定

### 2.2 VSL 训练（Variable Sequence Length）

设每步 token 预算为 b（固定超参数）。采样桶 D_i：

```
BS_i = b / 2^i        →    BS_i × 2^i = b  （总 token 恒定）
```

**每步 FLOPs 推导**：自注意力对序列长度是二次的。单序列 FLOPs ∝ (2^i)²。

```
FLOPs_i ∝ BS_i × (2^i)²
       = (b / 2^i) × 2^{2i}
       = b × 2^i
```

关键：每步 FLOPs 与序列长度**成正比**，而非二次方。因为 batch size 与序列长度互为倒数，消去了一个 2^i。

- D₈ (256): FLOPs ∝ b × 256
- D₁₁ (2048): FLOPs ∝ b × 2048
- D₁₃ (8192): FLOPs ∝ b × 8192（32× 差距）

传统固定 L=8192：每步恒为 b × 8192。DD 成本随采样桶变化。

### 2.3 Grow-P2 课程策略

训练过程中桶的采样权重由 α 控制：

```
w_i ∝ 1 / 2^{α·(i − i_min)}
```

α 随时间递减：

| 阶段 | α | 分布 |
|------|-----|------|
| 早期 | 大（如 4.0） | ~70% D₈–D₁₀, ~20% D₁₁–D₁₂, ~10% D₁₃ |
| 中期 | 中等 | 逐步拉平 |
| 后期 | 小（如 0.5） | 长桶频率增加 |

**加权平均 FLOPs**：

```
b × (0.70×256 + 0.20×2048 + 0.10×8192) ≈ b × 1408
加速比 = 8192 / 1408 ≈ 5.8 ≈ 6×
```

对比：传统 2K 训练每步 b × 2048。DD 加权 b × 1408 < b × 2048 → **DD 训练 8K 总成本 < 传统训练 2K**。

## 3. 消融实验（论文 Table 2）

| 配置 | 验证 Loss | 说明 |
|------|----------|------|
| Concat-and-Chunk（基线） | 3.55 | 固定 L=8192，无 VSL，无课程 |
| DD + Random 课程 | 3.35 | VSL 训练本身降 0.20 |
| DD + Linear 课程 | 3.22 | 有调度优于随机 |
| DD + Grow-P2 课程 | **3.18** | 最优。课程额外降 0.17 |

课程贡献约 46% 总增益（0.17/0.37）。**课程是 DD 的必需组件，非可选**。

## 6. 方法定位

**打破"序列长度必须固定"这一根本范式**。是四种方法中变革最深、最具前瞻性的方案。

**与 Best-Fit / ICLM 兼容**：桶内可再做装箱优化和语义排序。三者不互斥。

**局限**：当前验证在 1B 规模，百亿+参数未验证。需修改训练循环（变长 batch + 课程调度）。课程策略为必需组件。非 2 幂次 L 需额外 padding。
