# ① 随机拼接 (Concat-and-Chunk) — 基线方法

> **来源**: GPT-3 (Brown et al., 2020)、LLaMA/Llama-2 (Touvron et al., 2023)、PaLM (Chowdhery et al., 2022)、OPT (Zhang et al., 2022)、BLOOM (Scao et al., 2022) 等几乎所有主流 LLM 的默认做法
> **核心思路**: 将所有文档随机打乱后插入 EOS 分隔符首尾拼接为超长 token 流，再按固定上下文长度 L 切分为训练序列。

---

## 1. 算法流程

给定语料库 D = {d₁, ..., d_N} 和上下文长度 L。

**Step 1 — 随机置换**：对整个文档列表做 Fisher-Yates shuffle，产生随机排列 π = (π₁, π₂, ..., π_N)。

**Step 2 — EOS 拼接**：按 π 顺序依次 tokenize 每个文档，在末尾插入 EOS token，追加到全局缓冲区，形成一个超长 token 流：

```
T = tok(d_{π₁}) ∘ [EOS] ∘ tok(d_{π₂}) ∘ [EOS] ∘ ... ∘ tok(d_{π_N}) ∘ [EOS]
总长度 |T| = Σ l_i + N
```

**Step 3 — 定长切分**：从缓冲区头部开始，每满 L token 切出一段作为训练序列：

```
s_k = T[kL : (k+1)L],   k = 0, 1, 2, ...
序列总数 M = ⌊(Σ l_i + N) / L⌋
```

尾部不足 L token 的部分丢弃或 padding。每个训练序列严格 L token，零 padding，GPU 利用率 100%。

---

## 2. 截断率实测（Ding et al., ICML 2024，RefinedWeb）

| L | 截断率 | 平均每文档段数 |
|---|--------|---------------|
| 512 | ~85% | 3.2 |
| 1024 | ~72% | 2.1 |
| 2048 | ~60% | 1.6 |
| 4096 | ~45% | 1.3 |
| 8192 | ~30% | 1.15 |
| 32768 | ~12% | — |

增大 L 可减少截断，但 O(L²) 代价极大——L 从 2048 到 8192，截断率仅从 60% 降至 30%，成本变为 16 倍。问题未根本解决。

---

## 3. 三个根本缺陷（Ding et al., 2024）

**缺陷一：文档完整性被系统性破坏**

L=2048 下约 60% 文档以不完整形式参训。代码中变量定义与使用分离 → Undefined Name 错误。自然语言因果链切断。由于每 epoch shuffle 不同，截断位置持续变化，模型从未获得稳定的完整上下文。

**缺陷二：跨文档注意力噪声**

标准因果掩码不感知文档边界。序列中第 j 个文档的 token 无条件 attend 前 j−1 个文档的全部 token。前序文档随机选取，语义无关 → 注意力计算不提供预测信号。论文估计 15–30% 注意力计算被浪费。L 越大噪声占比越高。

**缺陷三：幻觉风险增大**

当 grounding context 与结论跨序列分离时，模型被训练为"在缺少充分上下文的条件下也做预测"。推理时即使给了完整上下文，模型仍可能忽略——训练经验告诉它"上下文经常不可靠"（context-based hallucination）。

**递进关系**：截断（根因）→ 跨文档噪声（表现）→ 幻觉（后果）

---

## 4. 历史定位

从 GPT-3 (2020, 175B, L=2048) 到 LLaMA (2023, 7B–65B)、PaLM (2022, 540B)，所有主流 LLM 统一采用此方案，长达五年。规模增益掩盖了拼接质量影响，传统评估（PPL/下游微调）对截断不够敏感，缺乏受控消融实验。直到 Ding et al. (2024) 做了"只改拼接方式、其他全不变"的对比实验——证明拼接策略对模型质量有显著影响。

---

## 5. 引出改进方向

随机拼接的三个缺陷为三种改进方向提供了靶点：

| 方向 | 方法 | 会议 |
|------|------|------|
| 算法层：最小化截断 | Best-Fit Packing | Amazon, ICML 2024 |
| 语义层：消灭噪声 | In-Context Pretraining | Meta, ICLR 2024 |
| 范式层：重新定义训练 | Dataset Decomposition | Apple, NeurIPS 2024 |

三者构成"无优化 → 长度优化 → 语义优化 → 范式反转"的递进谱系。不互斥，可组合使用。
